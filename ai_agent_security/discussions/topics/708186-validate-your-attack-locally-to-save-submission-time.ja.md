## 投稿

コンペ SDK に基づいて gpt-oss と gemma の両方を実行し、プロンプト攻撃を評価・スコア付けするローカル検証環境を作成しました。

このセットアップでは public guardrail に対するプロンプトの挙動しか観察できませんが、各プロンプトがどのように機能するかを詳細に分析するには非常に有用です。これにより、提出時に `interact()` を呼ぶ必要を完全に省略し、最終的なプロンプトをそのまま提出できます。

### 攻撃を評価する方法

1. このノートブックを開く: [AAS Local Validation](https://www.kaggle.com/code/llkh0a/aas-local-validation)
2. プレースホルダーのスクリプトを自分の攻撃スクリプトに置き換える。

現時点ではスターターノートブックのみ評価していますが、ローカルスコアは public リーダーボードのスコアとかなりよく相関しています。

**ローカル検証結果:**
```json
{
  "gpt_oss_public": 0.27,
  "gemma_public": 0.24,
  "local_public_mean": 0.255
}
```

**Public リーダーボードスコア:** 0.24
